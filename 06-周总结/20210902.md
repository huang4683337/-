## 进入表单页时报错

```
- 场景：
	进入表单页面时会弹出异常信息弹出框

- 原因
	表单涉及到新增和回填
	- 回填时需要根据根据上次提交的省市级别联信息进行回填处理。省市级联信息保存在一个数组中
	- 当是一个信息消息时，没有回填信息，就会报错没有 xx[0], 这样就触发了 catch ，出现了异常弹出框
	
- 结论
	涉及到数组的操作时需要对其做一个判空处理
```



## 发票状态展示按钮不显示

```
发票状态，0：未开票， 1：申请中， 2：审核通过， 3：审核驳回 ， 4：已开票
```

```
- 场景：
	已开票时不显示对应按钮

- 原因
	判断的是 审核通过 时展示按钮，在整个操作流程中没有触发到 已开票 的状态，当时就给过滤掉了
	
- 结论
	碰到对应多种状态时，要考虑全面，并且联系后端、PM 等了解各种状态的触发机制，然后再开发。
```
